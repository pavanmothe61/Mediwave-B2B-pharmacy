Day-wise Plan

Each working day is divided into 3 role-specific task columns. Student 1 = Frontend, Student 2 = Backend, Student 3 = Testing & Deployment. All three work in parallel.

Day 1
Week 1
01 June 2026 (Mon) – Introduction Session
The instructor explains the company background, project overview, problem statement, technology stack, team roles, 26-working-day plan, and review milestones for Pharmacy B2B Ordering Portal. No coding today; the goal is complete understanding.
Tasks – All 3 Students
Student 1 – Frontend
  • Watch and re-read the Introduction Section
  • Understand the frontend modules: Chat-style UI, FAQ screens, enquiry capture form, handover dashboard, conversation history
  • Sketch 3 screens on paper: input screen, workflow/assistant screen, dashboard
  • Start Day 1 logbook entry
Student 2 – Backend
  • Watch and re-read the Introduction Section
  • Understand backend responsibilities: Conversation APIs, knowledge base storage, prompt templates, fallback and handover logic
  • Write 5 backend questions about APIs, database, business rules, and deployment
  • Start Day 1 logbook entry
Student 3 – Testing & Deployment
  • Watch and re-read the Introduction Section
  • Research what testing means in a software project
  • Research GitHub and why version control is used
  • Create GitHub account if needed and start Day 1 logbook entry
Deliverable Groups formed. Logbook Day 1 entries written. All students understand project overview.
Day 2
Week 1
02 June 2026 (Tue) – Problem Statement + Abstract
Students document the business problem and write a project abstract for Pharmacy B2B Ordering Portal. These are Review 1 deliverables.
Tasks – All 3 Students
Student 1 – Frontend
  • Draft the problem statement in 4–5 sentences from the user interface perspective
  • List screens needed for Pharmacy/Doctor/User and Admin/MR/Warehouse Team
  • Write abstract focusing on user value and business impact
Student 2 – Backend
  • Draft the problem statement from technical/system perspective
  • List backend operations: create, read, update status, process logic, dashboard reports
  • Write abstract explaining APIs, database, and logic flow
Student 3 – Testing & Deployment
  • Create team GitHub repository
  • Set up folders: /frontend, /backend, /docs, /tests
  • Push README.md with title, company, team names, and tech stack
Deliverable Problem Statement draft. Project Abstract draft. GitHub repository created.
Day 3
Week 1
03 June 2026 (Wed) – Project Objectives + Tech Stack Setup
Students write objectives and prepare development environment for Pharmacy B2B Ordering Portal.
Tasks – All 3 Students
Student 1 – Frontend
  • Write 5 objectives from user/frontend perspective
  • Install VS Code and Node.js
  • Create frontend folder and first HTML/React screen
Student 2 – Backend
  • Write 5 objectives from backend/system perspective
  • Install Node.js Express or Python Flask setup
  • Create backend server with one test route
Student 3 – Testing & Deployment
  • Write 5 objectives from testing and quality perspective
  • Install Postman for API testing
  • Test backend Hello route and record result in test log
Deliverable 5 Project Objectives written. Development environment set up. First test documented.
Day 4
Week 1
04 June 2026 (Thu) – Use Case Design + Wireframes
Students design use cases and wireframes before coding major screens for Pharmacy B2B Ordering Portal.
Tasks – All 3 Students
Student 1 – Frontend
  • Draw wireframes for input form, processing/workflow screen, dashboard, detail page
  • Use paper or Figma free version
  • Upload wireframe images to /docs folder
Student 2 – Backend
  • Draw use case diagram with actors: Pharmacy/Doctor/User, Admin/MR/Warehouse Team, Admin
  • Write endpoint list: POST/create, GET/list, GET/detail, POST/process, GET/dashboard
  • Define request and response fields for each API
Student 3 – Testing & Deployment
  • Review wireframes and use case diagram for completeness
  • Write initial test cases for each use case
  • Push test case document to /docs
Deliverable Wireframes, use case diagram, API endpoint list, and test cases completed.
Day 5
Week 1
05 June 2026 (Fri) – Review 1 Preparation
All three students prepare Review 1 PPT and complete a mock presentation.
Tasks – All 3 Students
Student 1 – Frontend
  • Build slides: company intro, project title, user screens, wireframes
  • Keep slides clean and not overcrowded
  • Present frontend sections in team mock
Student 2 – Backend
  • Build slides: problem statement, proposed system, technology stack
  • Prepare simple explanation of backend approach
  • Present backend sections in team mock
Student 3 – Testing & Deployment
  • Build slides: objectives, abstract, GitHub repo link, test plan overview
  • Verify all documents are pushed to GitHub
  • Present testing sections in team mock
Deliverable Review 1 PPT slides completed and practiced. All docs pushed to GitHub.
